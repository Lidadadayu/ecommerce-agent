# 最终演示流程

## 启动系统

推荐一键启动：

```bash
python scripts/run_system.py --auto-port
```

系统会自动启动：

- FastAPI 后端
- Streamlit API 前端

## 演示顺序

### 1. 身份和领域边界

输入：

```text
你是谁？
```

说明系统不是通用聊天机器人，而是电商售前售后 Agent，当前领域是扫地机器人。

### 2. 售前推荐

输入：

```text
3000以内推荐一款扫拖一体机器人
```

展示：

- 售前需求识别
- 商品工具调用
- 结构化商品推荐
- 推荐理由

### 3. 型号详情

输入：

```text
RV4001 参数怎么样
```

展示：

- 商品 ID 识别
- 商品详情工具调用
- 参数展示

### 4. 型号对比

输入：

```text
对比 RV2001 和 RV4001
```

展示：

- 多型号识别
- 对比工具调用
- 参数横向比较

### 5. 故障诊断

输入：

```text
扫地机器人不回充怎么办
```

展示：

- 故障类型识别
- 自查步骤
- 处理建议
- RAG 知识库参考
- 售后建议

### 6. 高风险安全提醒

输入：

```text
机器人有烧焦味还能继续用吗？
```

展示：

- 安全风险识别
- 停止使用提醒
- 不自行拆机
- 联系售后检测

### 7. 订单与售后多轮

输入：

```text
帮我查一下 O202605010001 这个订单
```

然后输入：

```text
物流到哪了？
```

再输入：

```text
这个订单可以维修吗？
```

展示：

- 会话记忆继承订单号
- 订单工具
- 物流工具
- 售后判断工具

## 演示时强调的系统价值

1. 不是单纯问答，而是工具调用 + RAG + 多轮记忆。
2. 当前演示领域是扫地机器人，但底层是可配置领域包。
3. 售前和售后打通，覆盖从选购到维修。
4. 高风险售后场景有安全提醒和人工审核边界。
5. API 前后端分离，具备真实系统形态。
