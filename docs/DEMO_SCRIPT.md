# 项目演示脚本

## 项目定位

本项目是一个面向电商售前与售后场景的可配置 Agentic RAG 智能业务 Agent 系统。

当前演示领域是：

```text
扫地机器人与扫拖一体机器人
```

系统支持：

- 售前选购咨询
- 商品型号推荐
- 商品参数详情
- 多型号对比
- 故障排查
- 维护保养
- 售后政策解释
- 订单查询
- 物流查询
- 售后资格判断
- 售后工单创建
- 人工审核队列

## 推荐演示顺序

### 1. 身份与能力

用户输入：

```text
你是谁？
```

重点展示：

- 系统不是通用聊天机器人
- 当前领域是扫地机器人
- 支持售前、售后、订单、物流、人工审核

### 2. 售前推荐

用户输入：

```text
3000以内推荐一款扫拖一体机器人
```

重点展示：

- 路由到 `robot_vacuum_search`
- 查询 `products.json`
- 返回商品型号、参数和推荐理由

### 3. 多型号对比

用户输入：

```text
对比 RV2001 和 RV4001
```

重点展示：

- 路由到 `robot_vacuum_compare`
- 对比价格、吸力、续航、自动集尘、自动洗拖布等

### 4. 故障排查

用户输入：

```text
扫地机器人不回充怎么办
```

重点展示：

- 路由到 `robot_vacuum_knowledge_query`
- 走 RAG 混合检索
- 从故障排查知识库返回检测与修复建议

### 5. 维护保养

用户输入：

```text
边刷多久换一次？
```

重点展示：

- 走维护保养知识库
- 回答耗材维护周期

### 6. 订单与售后

用户输入：

```text
帮我查一下 O202605010001 这个订单
```

再输入：

```text
物流到哪了？
```

再输入：

```text
这个订单可以退货吗？
```

重点展示：

- 会话记忆继承订单号
- 工具调用订单/物流/售后
- 缺槽追问
- 人工审核规则
