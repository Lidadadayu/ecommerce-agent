入口统一与代码整理说明

这次补丁做了四件事：

1. 默认注册 /api/chat/stream
   修改 api/server.py，让流式接口成为默认 API 能力。
   以后不管你运行：
   python scripts/run_api.py
   还是：
   uvicorn api.server:app
   都应该有 /api/chat/stream。

2. 统一启动入口
   最终推荐只记一个命令：
   python scripts/run_system.py

   它会启动：
   - FastAPI 后端
   - SSE 流式接口
   - app_api_streaming.py 前端

3. 旧入口改成兼容入口
   app.py、app_api.py、app_api_humanized.py 都转发到 app_api_streaming.py。
   run_api_streaming.py、run_system_streaming.py、run_web_api.py 等也转发到最终入口。
   这样不会因为旧命令而启动到错误版本。

4. 修复一个用户隔离风险
   tools/business_tools.py 中 get_refund_progress() 现在会校验 customer_id。
   同时删除 get_operation_metrics() return 后面的死代码。

使用顺序：

1. 覆盖补丁文件
2. 运行：
   python scripts/entrypoint_health_check.py

3. 启动：
   python scripts/run_system.py --auto-port

4. 测试流式接口：
   python scripts/streaming_api_smoke_test.py --query "扫地机器人不回充怎么办"

旧文件不建议直接删除，因为可能有旧命令或旧测试引用。
现在它们已经变成兼容入口，不会再把你带到旧版本。
